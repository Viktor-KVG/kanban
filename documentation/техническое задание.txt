1) Многопользовательский режим
  - регистрация
  - логин
  - смена пароля
  - изменения настроек профиля
2) Бинарная система ролей (администраторы и обычные пользователи)
3) У пользователя может быть несколько досок
4) Возможность пригласить других пользователей на доску
5) CRUD для kanban досок
  - CREATE
  - READ
  - UPDATE
  - DELETE
6) kanban доска состоит из переменного числа столбцов (состояний задачи)
7) CRUD для столбцов
8) столбец может содержать в себе тикеты(задачи)
9) CRUD для тикета
10) тикет должен содеражть в себе:
  - название
  - описание
  - автор
  - deadline (опционально)
  - оценка времени выполнения (опционально)
  - приоритет (low/medium/high)
  - исполнитель
  - комментарии
11) CRUD для комментариев
12) Иметь возможность перемещать тикет как по столбцам в рамках доски, так и по разным доскам
13) Технологии: Docker + docker compose, postgresql 14, python3.11, pip3, fastapi, sqlalchemy, alembic, pydantic, pytest
